# 本番採用スタック決定（無料前提）

## 結論

- **本番候補（第一候補）**: `MapLibre GL JS + PMTiles（または自己管理ベクタタイル）`
- **バックアップ候補（第二候補）**: `Leaflet + OSM系タイル（必要に応じて有償/自己ホストへ移行）`

## 選定理由

- MapLibreは将来の表現拡張（スタイル、レイヤ制御、3D方向）に強い。
- PMTiles系は「無料で始めて、低コストでスケール」に相性が良い。
- Leafletは導入最速で実装負荷が低く、障害時の代替実装としても有効。

## フェーズ別方針

1. **PoC/検証**: Leaflet / MapLibre 並行比較（本リポジトリで切替可能化）
2. **初期リリース**: LeafletまたはMapLibreの軽量構成（トラフィック小）
3. **本格運用**: MapLibre + PMTiles/自己配信へ寄せる

## リスクと対策

- 公開無料タイル依存リスク  
  -> 早期に自己配信（PMTiles/CDN）手順を準備
- 地図表示ライブラリ変更コスト  
  -> アプリ側は `onDragEnd` など共通I/Fで吸収
- 利用規約変更リスク  
  -> 半年ごとに無料枠・商用可否・キャッシュ要件を再確認
