# Leaflet / MapLibre 最小PoC比較

同一アプリ内で、サイドバーから `Leaflet` / `MapLibre` / `Google` を切替可能に実装。

## 実装範囲

- 共通入力: 都道府県GeoJSON（既存フロー）
- 共通出力: `onDragEnd([lng, lat])` で国判定フロー接続
- 共通操作: 重心マーカードラッグ、初期位置へリセット

## 比較結果（今回のPoC）

- **Leaflet**
  - 実装量が少なく、導入が最速
  - OSMタイル前提で無料検証しやすい
  - 高度なスタイル/3D拡張はMapLibreより弱い
- **MapLibre**
  - ベクタタイル前提の拡張性が高い
  - 中長期の本命候補（PMTiles/自己配信と相性）
  - 初期セットアップはLeafletよりやや複雑

## 補足

- 現行Google版の「ポリゴンドラッグ補正」は高度機能のため、今回PoCでは「重心マーカードラッグ」で必須ユースケースを検証。
- 将来、MapLibre本採用時にポリゴンドラッグ補正ロジックを段階移植する。
